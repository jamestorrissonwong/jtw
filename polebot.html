PoleBot
I have worked in all aspects of this projects, ranging from troubleshooting hardware problems with our servo motors, to mechanical design of our module locking system, to the code that runs the robot.

Code
The code is currently being run in MATLAB, however I hope to move it to C++, or have it generated through SIMULINK once the vision is completed.
Vision
One of the more complex problems I have been working on has been integrating computer vision into our current system. We have refined the pole detection to the point where I believe we could move forward with integrating it into a controller. However, I don't believe this is possible given that we are supposed to be working remotely.
Mechanical Design
 I have done a lot of small design changes to various parts, however one of my original designs is the modular locking mechanism. The key constraint was that we couldn't add any additional motors to the robot, as we wanted the bot to remain light, and represent a "minimum viable product" therefore, we only had vertical and rotational movement. The lock that I designed fit these constraints, and was capable of supporting the robots through a vertical-to-horizontal pipe transition.
